# UX polish — PRD

Status: 🔄 in-progress
Lot: POLISH
Branch: feature/* or fix/* per ticket → PR → develop

## Problem Statement

Small, user-facing UX improvements to the tracker/UI surface as Walker is used day to day. They
deliver direct value but don't belong to any single feature lot and shouldn't be lost as ad-hoc
notes.

## Solution

A living lot (like TECH for tech debt): incremental UX-polish tickets are added here as they come up
and resolved individually, rather than the whole lot closing at once like a feature lot.

## Out of Scope

- Larger features that warrant their own lot.
- Non-functional/quality-only work (that's the TECH lot) and documentation-site content (DOCS lot).
