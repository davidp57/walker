# Technical debt — PRD

Status: 🔄 in-progress
Lot: TECH
Branch: fix/tech-* or chore/tech-* per ticket → PR → develop

## Problem Statement

Cross-cutting technical-debt items (refactors, quality follow-ups, consistency fixes) surface during
work on other lots but don't belong to any single feature. Rather than force-fitting them into an
unrelated feature lot or losing them as ephemeral notes, this lot is a standing home for them.

## Solution

A living lot: tickets are added here as debt is identified (e.g. from code review follow-ups) and
removed/archived individually once resolved, rather than the whole lot closing at once like a
feature lot does.

## Out of Scope

New features or user-facing behavior — this lot is exclusively non-functional/quality work.
