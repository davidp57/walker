# I18N — Bilingual user-facing documentation (archived)

Status: ✅ done
Branch: chore/i18n-* / feature/* per ticket → PR → develop

## Summary

Makes the published MkDocs site (`docs-site/`) bilingual: English primary (the default, served at the
site root, and the fallback for any untranslated page) and French a fully-translated secondary, via
the `mkdocs-static-i18n` plugin (suffix structure, `.fr.md` siblings) with a language switcher in the
Material header. Scope was `docs-site/` only — a deliberate, documented exception to CLAUDE.md's
"all repo artifacts in English" rule; internal dev docs, the README, code, and UI copy stay
English-only.

## Verified against

- `mkdocs build --strict` exits 0 with no warnings; all 7 French pages build under `site/fr/`, the
  English pages carry `hreflang` alternates for `en`/`fr` (the language switcher), and French content
  renders. The `docs.yml` CI job runs the same build (with the plugin installed via
  `pip install ".[docs]"`) on push to `main`.

## Notes

- Structure is `docs_structure: suffix` (`index.md` = English default, `index.fr.md` = French), so
  English filenames/paths are unchanged and the change is purely additive.
- Product names, Timesheet-system code names, CLI commands, and env vars are kept verbatim in the
  French pages (per CLAUDE.md).
